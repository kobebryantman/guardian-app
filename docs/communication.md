# Guardian 通信架构文档

覆盖三个端：**管控服务器**、**教师端**（浏览器 + Electron）、**学生端**（Electron）

> 详细规范见 `docs/protocol.md`（通信协议）和 `docs/tech-stack.md`（技术栈方案）。

---

## 1. 系统拓扑

```
┌─────────────────────────────────────────────────────────────────┐
│                     管控服务器 (guardian-server)                    │
│                                                                  │
│  HTTP Layer               WS Layer           Storage Layer       │
│  ┌──────────┐       ┌───────────────┐       ┌───────────────┐   │
│  │ REST API │       │  WS Gateway   │       │  PostgreSQL   │   │
│  │ (Express)│       │  /guardian-ws │       │  + Redis      │   │
│  │          │       │               │       │               │   │
│  │ 教师认证  │       │ 消息路由       │       │ 持久化 + 缓存  │   │
│  │ 房间 CRUD │       │ 房间管理       │       │               │   │
│  │ 静态页    │       │ 指令下发       │       │               │   │
│  └────┬─────┘       └──────┬────────┘       └───────────────┘   │
└───────┼────────────────────┼───────────────────────────────────┘
        │ HTTP               │ WS
        │                    │
   ┌────┴────────┐    ┌──────┴──────────┐    ┌────────────────┐
   │ 教师端(浏览器)│    │ 教师端(Electron) │    │ 学生端(Electron) │
   │             │    │                 │    │                │
   │ HTTP CRUD   │    │ HTTP + WS       │    │ WS 直连         │
   │ WS 订阅通知  │    │ (实时监控)       │    │ (心跳+守卫循环)  │
   └─────────────┘    └─────────────────┘    └────────────────┘
```

所有端都是 client，角色不同。Server 是纯中枢，做认证、路由、状态管理。

---

## 2. 通信总览

| 链路 | 协议 | 方向 | 说明 |
|------|------|------|------|
| 教师端 ↔ 管控服务器 | **HTTP REST** (JSON) | 双向 | 登录注册、房间 CRUD、批量操作 |
| 教师端 ↔ 管控服务器 | **WebSocket** (JSON) | 全双工 | 订阅房间状态、实时推送 |
| 学生端 ↔ 管控服务器 | **WebSocket** (JSON) | 全双工 | 心跳上报、违规上报、接收指令 |

教师端两条通道：HTTP 做 CRUD，WS 做实时推送。

## 4. 技术栈

见 `docs/tech-stack.md`，核心选型：

| 层面 | 小规模方案 | 中大规模方案 |
|------|-----------|-------------|
| 持久化 | SQLite | PostgreSQL |
| Runtime 状态 | In-Memory Map | Redis |
| WS 广播 | Map.forEach + ws.send() | Redis Pub/Sub |
| 违规写入 | 直接写 | 攒批缓冲区 → batch INSERT |
| 进程模型 | 单进程 | Cluster / 多服务器 |

## 5. 核心流程

### 学生上机

```
学生端                          管控服务器                    教师端
  │                                │                          │
  │  WS /guardian-ws               │                          │
  │ ──────────────────────────────►│                          │
  │◄── { welcome, clientId }       │                          │
  │                                │                          │
  │  { bind, roomCode,            │                          │
  │    studentId, studentName }    │                          │
  │ ──────────────────────────────►│                          │
  │                                │ 验证 roomCode             │
  │◄── { bind-ack, ok, roomId }    │                          │
  │                                │                          │
  │  { heartbeat } 每 5s           │  WS 推送新上线            │
  │ ──────────────────────────────►│ ─────────────────────────►│
  │◄── { heartbeat-ack }           │                          │
```

### 教师远程管控

```
教师端                          管控服务器                    学生端
  │                                │                          │
  │  POST /api/rooms/:id/start     │                          │
  │ ──────────────────────────────►│                          │
  │                                │ WS { toggle-guard,      │
  │                                │      enabled: true }     │
  │                                │ ───────────────────────►│
  │                                │                main.js: 启动守卫
  │  WS 推送: 子机状态变更          │                          │
  │◄──────────────────────────────│                          │
```

### 违规检测

```
学生端 main.js                     管控服务器                   教师端
  │                                │                          │
  │  PowerShell 扫描进程            │                          │
  │  白名单比对 → 违规               │                          │
  │                                │                          │
  │  WS { heartbeat, violations }  │                          │
  │ ──────────────────────────────►│                          │
  │                                │ 内存缓存 → 攒批缓冲区     │
  │                                │ 每 10s / 100 条 → 写 DB  │
  │                                │                          │
  │                                │ WS 推送违规通知           │
  │                                │ ───────────────────────►│
```
